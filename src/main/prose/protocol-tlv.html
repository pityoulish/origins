Binary Protocol

A protocol for implementing the Message Board with TLV-encoded binary messages exchanged over socket connections.

Overview...

  • Message Board
  • client and server
  • TLV

Connections

...

  • Server listens, clients connect.
  • Client sends request, receives response.
  • Server may close connection after response, or keep it open.
  • If kept open, client may send another request, or close.

TLV

...

  • Type - Length - Value
  • primitive or constructed (nested TLVs)

Primitive TLV

0xC0  -  Text
free-form text for consumption by humans
0xC1  -  Originator
username or system
0xC2  -  Timestamp
timestamp in ISO format
0xC3  -  Marker
position marker in the message stream
0xC4  -  Limit
maximum batch size
0xC5  -  Missed
indicator for missed messages; present/absent or value T/F?

Constructed TLV

0xE0  -  Message
Originator, Timestamp, Text; order of elements?
0xE1  -  Info Response
Text
0xE2  -  Error Response
Text
0xE3  -  List Messages
Limit, optional Marker
0xE4  -  Message Batch
Marker, optional Missed, optional list of Message; order of elements?
0xE5  -  Put Message
Ticket, Text
0xE6  -  Obtain Ticket
Originator (username)
0xE7  -  Ticket Grant
Ticket
0xE8  -  Return Ticket
Ticket
0xE9  -  Replace Ticket
Ticket
