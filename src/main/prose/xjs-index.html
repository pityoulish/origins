Exercise: Message Board Client with Sockets

During this exercise, you will get a glimpse of what it means to implement a low-level protocol interface on top of socket connections.

Overview... it's about the Message Board and Java Sockets. With a Binary Protocol.

Test Sequence

Different from Tutorial, because of dependencies between the commands. Need a ticket for some. Need enough messages for others. Suggestion...

  • list messages, without marker
  • obtain ticket
  • put message; verify with list messages
  • list messages, with marker
  • return ticket; verify with put message
  • replace ticket; have to obtain a new one first
  • Error cases are not explicitly covered by this test sequence. Can you think of non-error cases that are not covered?

Test, Fix, Build, repeat
