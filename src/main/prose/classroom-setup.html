Classroom Setup

All Pityoulish classroom exercises share a common setup: students connect to the instructor's server; server output appears on a public screen. Below are instructions to establish this setup and to run an exercise, including preparation and debriefing.

The first two sections are about technical aspects, the other two about organization.

  1. What You Need in hardware and software
  2. Rehearse so you know what to do
  3. Schedule well in advance
  4. Lessons Learned for yourself and others

What You Need ¶

Server Machine
Even a Netbook will have sufficient computing power and memory for running the server software. The machine needs video output, so that you can connect it to a projector or other public display. Check which connector (HDMI, VGA), cables, or adapters you need.
Server Runtime
As of 2016, the required server runtime is a Java Virtual Machine (JVM), or better a Java Development Kit (JDK). Future enhancements might add components for Node.js, Python, or other runtimes. If your server machine is an x86 architecture running Linux, Windows, or MacOS, these runtimes are readily available. If you're on another architecture, for example a Raspberry Pi, better double-check the availability of the runtimes you require.
Network
A wireless access point, without Internet connection, should be enough to serve a dozen groups of students or more.¹ When working in groups, students can still do online research from a different machine than the one connected to the exercise network. If they wanted to cheat by accessing the Pityoulish source code, you couldn't stop them anyway.
If you prefer to use an on-site network in the classroom, either wired or wireless, verify that there are no firewalls that would prevent the student machines from connecting to your server, or to eachother.

¹) In 2016, the corporate laptop of one student acted up though, complaining about "limited availability" of the network.

Web Server
The Pityoulish exercises are packaged for presentation on a web server. If the exercise network has no Internet connection, you should run a web server on the server machine. Otherwise, you can upload the material on any public web server. The content students need to access are the exercises themselves, and required documentation such as the standard JavaDocs. The exercises can also be downloaded as a zip file and extracted locally, for browsing with a file: URL.
Client Machine
A dedicated client machine is required only for the rehearsal, but you MUST rehearse. Some network related problems, like a wrong hostname or IP address in an RMI stub, will only occur if you run the client code on a different machine than the server component.

Rehearse ¶

Described here is the Tech Rehearsal, to sort out problems with cables, network, and the like. Once completed, you should be fine with the technical aspects until you change the classroom or some of your equipment. But there are also Content Rehearsals specific to each exercise. You need to run those, too.

Network ¶

  1. If you are running your own wireless access point, start it up. What's the WLAN name and password that students need to connect? Do you need to configure the firewall or other security options of the access point?
  2. Connect the server machine to the network, wired or wireless. What's the hostname or IP address that students need to contact?
  3. What's the URL to the exercise and other information that students need? Do you have a landing page from which all the rest is linked? If you provide the information on a local web server, start it now.
  4. Start the exercise-specific server in a terminal. What's the port number that students need to connect to? This is actually part of the Content Rehearsal, see the Instructor's Notes of the exercise for details.
  5. Connect a client machine to the network. This must be a different machine, so that you are actually testing the network rather than machine-local connections.
  6. From the client machine, access the web server. Make sure that the exercise description as well as other required information is accessible.
  7. From the client machine, access the exercise-specific server. Again, this is part of the Content Rehearsal, see the Instructor's Notes of the exercise for details.

Display ¶

  1. Connect the server machine to the projector, or another public display in the classroom. Do you have to adjust desktop settings? Mirror the display, extend the desktop, or use only the external display?
  2. Move the terminal of the exercise-specific server to the public display. If you need other windows on the public display, open them now. For example, a terminal running a Follow-the-Board client. This is part of the Content Rehearsal, see the Instructor's Notes of the exercise for details.
  3. Make sure that each terminal is at least 80 columns wide. Arrange the windows. Adjust the font size and colors, so that output is readable from the back row.
  4. How will you provide the network information and URL to the students? Whiteboard, blackboard, flipchart? They might be obstructed by the projection screen. You can't use the server terminal, information would scroll off.

Schedule ¶

At which points during the term are you planning the classroom exercises? The students need to work through the Tutorial first. Is there additional reading or preparation you expect of them? For programming in general, or for specific exercises? Make sure to hand out such assignments some time in advance.

When do you intend to teach non-programming topics related to the classroom exercises? Will you use the exercises as introduction, put the programming after the theory, or interleave both in the same classroom session?

When can you make time for your own rehearsals? Do not underestimate this! You must rehearse.

On the days with classroom exercises, plan a huge slot for discussions. You might need it because students don't progress as fast as you would have thought, or are missing some knowledge that you assumed implicitly. Still, prepare an optional topic as well, in case everything goes smoothly and you have some time left.

How will you form work groups of two or three students? Those sitting next to eachother will group easily, but you might end up with some students working alone. Do you want the same groups for each programming session, or do you want to mix groups over the term? If students need to change seats, plan in some extra time for replugging the laptops.

Lessons Learned ¶

Gather feedback from the students immediately after the classroom exercises. And again at the end of the term, with hindsight. Record their feedback as well as your own observations in writing, for the next time.

Did you learn something that might be useful to others? Please provide feedback, positive or negative, to the Open Source project at https://github.com/rolandweber/pityoulish/. Just open an issue and share your experience.

If you want to modify an exercise or add your own, fork the project. You're welcome to contribute improvements or additions back upstream, but you're not required to. The project sources are in the public domain.
